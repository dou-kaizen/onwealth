# Port Swagger/OpenAPI Setup with Scalar UI and Env-Gated Exposure

**Date**: 2026-05-04 02:05
**Severity**: Medium
**Component**: `apps/api/src/config/swagger.config.ts`, `@onwealth/platform/problem-details`, `apps/api/src/main.ts`
**Status**: Resolved

## What Happened

Ported `@nestjs/swagger` + Scalar API Reference from the boilerplate into `apps/api`. Three surfaces exposed: `/swagger` (Swagger UI), `/docs` (Scalar), `/swagger-json` + `/openapi.yaml` (codegen). One novel addition not in the boilerplate: `ENABLE_SWAGGER` env gate using `z.enum(['true','false'])` strict whitelist so the routes never mount in production unless explicitly opted in.

Two substantive side effects: `ProblemDetailsDto` was converted from interface to class (required for `@ApiProperty` decorators and `extraModels` schema injection), and `addDefaultErrorResponses()` auto-injects a `default: $ref ProblemDetailsDto` response on every OpenAPI operation so FE codegen emits one error type instead of N status-keyed branches.

Shipped in `7d1685d` (163 insertions, 3 files), docs synced in `8fd43a5` which also removed the stale CLS interceptor journal.

## The Brutal Truth

The red team review caught 14 issues from 21 raw findings before a single line was written — two of them were Critical. Finding #1 was a wrong package name (`@onwealth/backend` instead of `@onwealth/api`) appearing 10 times across 4 phase files. That would have made every `pnpm --filter` command in the phase silently succeed without building anything. Finding #2: `helmet()` default CSP blocks Swagger UI and Scalar inline scripts, meaning curl smokes pass but the browser renders a blank page. Both would have burned time on "why does the smoke say green but nothing works."

The `ENABLE_SWAGGER` env gate is a deliberate deviation from the boilerplate, which exposes Swagger unconditionally. That's the right call — but it introduced a new failure mode: Swagger routes bypass `ThrottlerGuard` entirely because `SwaggerModule.setup` and `app.use('/docs', ...)` are Express middleware, outside the NestJS guard pipeline. Accepted and documented; mitigation deferred to when staging becomes publicly accessible.

The interface-to-class conversion of `ProblemDetailsDto` was the trickiest structural change. All 3 filter consumers use `import type` — so the class form works transparently as a type. But the barrel in `packages/platform/src/problem-details/index.ts` needed `export` not `export type` because a class is a value, not just a type. Missing that would have caused a runtime "not a constructor" error that TypeScript would not have caught.

## Technical Details

- **`apps/api/src/config/swagger.config.ts`** (113 lines): `DocumentBuilder` chain, `addDefaultErrorResponses()` mutates the document in place iterating all paths × all HTTP methods, `SwaggerModule.setup('swagger', ...)` + `app.use('/docs', apiReference({...}))`.
- **`ENABLE_SWAGGER` in `env.schema.ts`**: `z.enum(['true', 'false'])` — rejects `'1'`, `'yes'`, `'on'`. Invalid value = loud boot failure, not silent fallback. Default: `true` non-prod, `false` prod (via `??` fallback in `main.ts`).
- **Server URL**: threaded from `API_BASE_URL` env through `setupSwagger(app, configService)` — hardcoded `localhost:3000` would break FE codegen at any non-local environment.
- **CSP split**: when `swaggerEnabled`, helmet is reconfigured to allow `'unsafe-inline'`, `'unsafe-eval'`, `cdn.jsdelivr.net` for Scalar's CDN-loaded scripts. When `swaggerEnabled === false` (production), strict default helmet CSP is unchanged. The Helmet path-match note from the hardening journal applies here too: `app.use('/swagger', mw)` does NOT match `/swagger-json`; JSON endpoint stays under strict CSP regardless.
- **`addDefaultErrorResponses` dead branch fix** (red team finding #11): original boilerplate version had `responses.default["$ref"]` branch that masked malformed responses; removed, single direct path used instead.
- **YAML endpoint**: `/openapi.yaml` exposed via a separate `app.get('/openapi.yaml', ...)` handler — not auto-generated by SwaggerModule, added manually to match boilerplate.

## Root Cause Analysis

No failures to root-cause — this was a clean port with a rigorous pre-implementation red team pass. The 10-occurrence wrong package name is a template artifact from the plan scaffold tool using a wrong default; that's a tooling bug, not a design error. The CSP gap is fundamental to how Swagger UI and Scalar work (inline scripts, CDN assets) and the boilerplate simply never addressed it because it had no env gate.

## Lessons Learned

- **Run red team review on plan files, not just code.** Two Critical findings (`@onwealth/backend` package name, CSP) were caught before implementation, not after. The cost of fixing in plan text is near-zero. The cost of debugging in running code is measured in hours.
- **`z.enum(['true','false'])` is the correct pattern for boolean env vars.** `z.boolean()` applied to a string env var silently coerces everything; `z.coerce.boolean()` accepts `'0'` as truthy. Strict whitelist with loud failure on invalid input is safer.
- **Interface → class conversion requires updating barrel export style.** `export type { ProblemDetailsDto }` → `export { ProblemDetailsDto }`. TypeScript won't catch missing `export` on a class; it will compile and fail at runtime when `extraModels: [ProblemDetailsDto]` tries to call the constructor.
- **Express middleware routes bypass NestJS guard pipeline.** `SwaggerModule.setup` and `app.use(...)` are not in the NestJS request lifecycle. `ThrottlerGuard`, `AuthGuard`, and any `APP_GUARD` provider are invisible to them. Design accordingly.
